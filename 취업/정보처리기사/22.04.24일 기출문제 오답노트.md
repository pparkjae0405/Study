트랜잭션의 특성
- 원자성(Atomicity) :  완전히 수행완료되지 않으면 전혀 수행되지 않아야 함
- 일관성(Consistency) : 고정 요소는 수행 전후에 같아야 함
- 격리성(Isolation) : 실행 시 다른 트랜잭션의 간섭X
- 영속성(Durability) : 완료 결과가 DB에 영구히 기억됨

- Best Effort원칙에 따른 전송기능 제공
- 헤더 체크섬만 제공

LRU(Least Recently Used) 알고리즘
- 가장 오랫동안 사용하지 않은 페이지를 교체.
- 결함 : 주 기억장치의 페이지가 변동되는 것

사용자 수준에서 지원되는 스레드(thread)가 커널에 비해 가지는 장점
- 커널 모드로의 전환 없이 스레드 교환 가능, 오버헤드 적음

- 내용 결합도(Content Coupling) : 한 모듈이 다른 모듈의 내부기능&내부 자료를 참조
- 스탬프 결합도(Stamp Coupling) : 두 모듈이 동일한 자료구조 조회
- 공통 결합도(Common Coupling) : 공유되는 공통 데이터영역을 여러 모듈이 사용
- 제어 결합도(Control Coupling) : 한 모듈이 다른 모듈의 상세 처리 절차 인지하여
  통제 or 처리 기능이 두 모듈에 분리되어 설계된 경우

모듈화(Modularity)
- 시스템을 모듈로 분할하면 각각의 모듈을 별개로 만들고 수정가능, 좋은 구조
- 모듈 간 결합도가 약해야 독립적
- 모듈 내 구성요소들 간 응집도가 강해야 좋은 설계
- 응집도 : 모듈이 독립적인 기능으로 정의되어 있는 정도
- 결합도 : 모듈과 모듈 사이 상호의존 또는 연관 정도
