# 22.03/05 정보처리기사 기출문제 오답노트

1과목 : 소프트웨어 설계
```
- 하향식 설계 : 인터페이스가 정의되어있어 통합이 간단, 레벨이 낮은 데이터 구조 세부사항은 초기 단계에서 필요
- 상향식 설계 : 최하위수준 모듈을 설계하고 결합하여 검사, 인터페이스 성립되어있지 않을 시 기능추가가 어려움

- 목업(Mockup) : 디자인, 사용방법, 평가 등을 위해 실제화면과 유사하게 만든 
  정적인 형태의 모형, 시각적으로만 배치하여 실제구현X

UML 다이어그램의 분류
- 정적 구조 : 클래스, 객체, 패키지, 컴포넌트, 복합구조, 배치
- 동적 구조 : 유스케이스, 상태, 활동, 시퀀스, 통신, 상호작용, 타이밍

객체지향 분석기법
- Rumbaugh : 객체->동적->기능 모델링(모형)
- Booch : 클래스&객체 분석 및 속성과 연산 정의
- Jacobson : 유스케이스 강조
- Coad와 Yourdon : E-R 다이어그램 사용, 객체 행위 모델링
- Wirfs-Brock : 분석과 설계 간 구분 X
```
2과목 : 소프트웨어 개발
```
- 테스트 드라이버(Test Driver) : 단위 테스트 시 하위 모듈을 호출하고 파라미터를
  전달하는 가상의 모듈, 상향식 테스트 시 사용

- 프로젝트 관리 : 주어진 기간 내에 최소비용으로 사용자를 만족시키는 시스템 개발

- 정형 기술 검토(FTR) : 의제 제한, 논쟁&반박 제한, 문제영역 명확히 표현, 참가자 제한

자료 구조의 분류
- 선형 구조 : 스택, 큐, 데크, 리스트
- 비선형 구조 : 트리, 그래프

- JSON, XML, YAML : 인터페이스 간 통신을 위해 이용되는 데이터 포맷

- 프로그램 설계도 NS-Chart : 3가지 구조만으로 논리 표현, 화살표X, 입구출구 1개

- Quick Sort(퀵 정렬) : 분할정복 알고리즘, 피벗을 사용하여 n(n-1)/2회 비교 수행 정렬

- 화이트박스 테스트 : 기초경로, 조건, 루프, 데이터 흐름 검사
- 블랙박스 테스트 : 동등분할, 경계값, 원인-효과, 오류 예측, 비교 검사

ISO/IEC 25000
- 2500n 개요 및 품질관리
- 2501n 품질모델
- 2502n 품질측정
- 2503n 품질요구
- 2504n 품질평가
- 2505n 확장분야

- 코드 인스펙션 : 코드 품질 향상 기법, 정적 테스트, 산출물을 근거로 수행하여
  눈으로 확인, 결함+표준준수+효율성 이슈 검사
```

3과목 : 데이터베이스 구축
```
- DELETE : 테이블 내 튜플들만 삭제
- DROP : 테이블 자체 삭제

- 제 1 정규형 : 모든 속성의 도메인이 원자값으로만 구성
- 제 2 정규형 : 1에 속하고, 기본키가 아닌 모든 속성이 기본키에 완전함수 종속됨
- 제 3 정규형 : 2에 속하고, 기본키가 아닌 모든 속성이 기본키에 이행적 함수종속X
- BCNF(보이스/코드) 정규형 : 모든 결정자가 후보키
- 제 4 정규형 : BCNF 만족, 다치 종속 제거
- 제 5 정규형 : 4 만족, 후보키를 통하지 않는 조인 종속 제거

- ∃ : 존재한다(There exist)
- ∈ : t가 r에 속함( t ∈ r )
- ∀ : 모든 것에 대하여(for all)
- ∪ : 합집합

- Cardinality = 행 (행카)
- Degree(차수) = 열 (열차)

- 이행적 종속 : A->B, B->C일 때 A->C만족, 이행적 종속 제거는 2 -> 3 정규형

- CREATE TABLE : 속성 NOT NULL 여부 지정, 기본 키를 구성하는 속성 지정, CHECK 제약조건 정의

DDL(Data Define Language, 데이터 정의어)
- SCHEMA, DOMAIN, TABLE, VIEW, INDEX 정의, 변경, 삭제 시 사용
- CREATE, ALERT, DROP
DML(Data Manipulation Language, 데이터 조작어)
- 저장된 데이터를 실질적으로 처리하는 데 사용
- SELECT, INSERT, DELETE, UPDATE
DCL(Data Control Language, 데이터 제어어)
- 데이터의 보안, 무결성, 회복, 병행 수행 제어 등을 정의하는 데 사용
- COMMIT, ROLLBACK, GRANT, REVOKE
```
4과목 : 프로그래밍 언어 활용
```
IPv4
- 32비트 주소, 유니/멀티/브로드캐스트 사용, 클래스 별 네트워크/호스트 주소길이 다름, 호스트 주소 자동 설정
IPv6
- 128비트 주소, 인증/기밀/무결성 지원(보안성 강화), 유니/애니/멀티캐스트 사용,
  주소 자동 설정, 16비트씩 8부분의 16진수 표기, 2^128개 주소 표현 가능,
  등급/서비스 별로 패킷 구분 가능, 확장기능으로 보안기능 제공

자바의 예외(exception)
- 실행 시간 동안 오동작/결과 악영향을 미칠 수 있는 경우
- 배열의 인덱스가 그 범위를 넘어서는 경우
- 존재하지 않는 파일을 읽으려고 하는 경우

- ICMP(Internet COntrol Message Protocol) : TCP/IP 계층 구조에서의 IP 동작 과정에서 
  전송 오류가 발생하는 경우를 대비해 오류 정보를 전송하는 목적으로 사용하는 프로토콜

UNIX 시스템의 쉘(shell)의 기능
- 사용자 명령 해석& 커널로 전달
- 반복적인 명령 프로그램을 만드는 프로그래밍 기능 제공
- 쉘 프로그램 실행을 위해 프로세스&메모리 관리
```

5과목 : 정보시스템 구축관리
```
- Parsing : 프로그램을 실제 실행할 수 있는 내부 포맷으로 분석&변환
- LAN Tapping : LAN 신호를 직접 자신에게 끌어오는 공격
- Switch Jamming : 스위치의 기능이 방해, 더미 허브처럼 작동
- FTP Flooding : DoS공격, 다량의 SYN 패킷을 보내 백로그 큐를 가득 채우는 공격

V모델
- 세부적인 테스트 과정으로 구성, 신뢰도 높은 시스템 개발
- 개발, 검증 사이의 관계를 명확히 들어낸 폭포수 모델의 변형
- 폭포수는 산출물 중심, V는 작업&결과의 검증

블루투스(Bluetooth)공격
- 블루버그(Bluebug) : 취약한 연결 관리를 악용
- 블루스나프(BlueSnarf) : 취약점을 이용하여 장비의 파일에 접근
- 블루프린팅(BluePrinting) : 공격 장치의 검색 활동
- 블루재킹(BlueJacking) : 스팸처럼 명함을 익명으로 퍼트리는 것

- DES : 암호화 알고리즘, 7비트 + 오류 검출 정보 1비트, 8*8 = 64비트

- wtmp : 리눅스 시스템에서 사용자의 성공한 로그인/아웃 정보기록, 시스템 종료/시작시간 기록

- 낙관치, 비관치, 기대치 : 상향식 비용 산정 기법 중 LOC(원시 코드 라인 수)기법에서 예측치를 구하기 위해 사용하는 항목

- DPI(Deep Packet Inspection, 내부 패킷 분석) : OSI 7Layer 전 계층 프로토콜 + 패킷 내부
  콘텐츠를 파악하여 침입 시도, 해킹 등을 탐지하고 트래픽을 조정하기 위한 패킷 분석 기술
```
