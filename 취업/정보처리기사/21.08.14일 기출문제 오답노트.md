```
무결정 규정
- 데이터가 만족해야 할 제약 조건, 규정을 참조할 때 사용하느 식별자 등의 요소가 포함
- 무결성 규정의 대상으로는 도메인, 키, 종속성 등이 있음
- 릴레이션 무결성 규정 : 릴레이션 조작과정에서의 의미적 관계 명세
- 개체 무결성 : 기본 키는 NULL이 될 수 없음
- 참조 무결성 : 외래 키는 참조할 수 없는 값을 가질 수 없음
- 도메인 무결성 : 특정 속성값은 그 속성이 정의된 도메인에 속한 값이여야 함

- 카티션 프로덕트 : 두 테이블 곱집합, 2중 for문같은 것

물리적 데이터베이스 설계
- 효율적인 방법으로 데이터를 저장하기 위한 목적
- 트랜잭션 처리량, 응답 시간, 디스크 용량 등 고려
- 저장 레코드의 형식, 순서, 접근 경로같은 정보를 사용하여 설계
논리적 데이터베이스 설계
- 트랜잭션의 인터페이스 설계

- HAVING SQL문 : GROUP BY와 함께 사용되며, 그룹에 대한 조건 지정

- 관계 대수 연산 : Select, Project, Join, Division

- 슈퍼 키 : 유일성을 만족하는 속성 또는 속성들의 집합
- 후보 키 : 유일성& 최소성 만족하는 속성 또는 속성들의 집합
- 대체 키 : 기본 키로 선택되지 못한 후보키
- 외래 키 : 다른 릴레이션의 기본 키를 참조하는 속성 또는 속성들의 집합

로킹 단위(Locking Granularity)
- 로킹 단위↑ -> 로크의 수↓ -> 병행성 수준↓, 기법 간단
- 로킹 단위↓ -> 로크의 수↑

- 제 1 정규형 : 모든 속성의 도메인이 원자값으로만 구성
- 제 2 정규형 : 1에 속하고, 기본키가 아닌 모든 속성이 기본키에 완전함수 종속됨
- 제 3 정규형 : 2에 속하고, 기본키가 아닌 모든 속성이 기본키에 이행적 함수종속X
- BCNF(보이스/코드) 정규형 : 모든 결정자가 후보키
- 제 4 정규형 : BCNF 만족, 다치 종속 제거
- 제 5 정규형 : 4 만족, 후보키를 통하지 않는 조인 종속 제거

- X -> Y : Y는 X에 함수 종속을 표현
```

```
응집도
- 시간적 응집도(Temporal Cohension) : 서로 다른 기능을 같은 시간대에 함께 실행하는 경우
- 논리적 응집도(Logical Cohension) : 같은 범주에 속하는 기능끼리 묶인 경우(새글, 불러오기, 저장하기, 다른이름으로 저장)
- 우연적 응집도(Coincidental Cohension) : 뚜렷한 관계 없이 묶인 경우
- 순차적 응집도(Sequential Cohension) : 이전의 명령어로부터 나온 출력 결과를 그 다음 명령어의 입력 자료로 사용하는 경우

오류 제어에 사용되는 자동반복 요청방식
- 정지-대기(Stop-and-wait ARQ)
- Go-back-N ARQ
- 선택적 재전송(Selective-Repeat ARQ)
- 적응적(Adaptive ARQ)

IP address
A class : 0 ~ 127
B class : 128 ~ 191
C class : 192 ~ 223
D class : 224 ~ 239 (멀티캐스트용)
E class : 240 ~ 255 (연구용)

페이지 교체 알고리즘
- OPT(Optimal) : 앞으로 가장 오랫동안 사용되지 않을 페이지 교체
- FIFO(First In First Out) : 먼저 들어온 페이지가 먼저 나감
- LRU(Least Recently Used) : 가장 오랫동안 사용되지 않은 페이지 교체
- LFU(Least Frequently Used) : 참조 횟수가 가장 작은 페이지 교체
- MFU(Most Frequently Used) : 참조 횟수가 가장 많은 페이지 교체
- NUR(Not Used Recently) : 최근에 사용하지 않은 페이지 교체

- C언어 변수선언 중 else는 예약어로 변수명 사용 X
```

5과목 : 정보시스템 구축관리
```
침입 탐지 시스템
- 이상 탐지 기법(Anomaly Detection) : 잘 알려지지 않은 공격 탐지
- HIDS(Host-Based Intrusion Detection) : 운영체제에 설정된 계정에 따라 사용자의 접근, 작업 기록&추적
- NIDS(Network-Based Intrusion Detection System) : Snort
- 오용 탐지 기법 (Misuse Detection) : 잘 알려진 시스템의 취약점을 공격하는 패턴의 침임을 탐지

정보 시스템 접근 제어 방법
- 강제 접근 통제(Mandatory Access Control) : 주체와 객체의 등급을 비교하여 접근 권한 부여
- 임의 접근 통제(Discretionary Access Control) : 접근하는 사용자의 신원에 따라 접근 권한 부여
- 사용자 계정 컨트롤(User Access Control) : 프로그램에서 관리자 수준의 권한이 필요한 
  작업을 수행할 시 사용자에게 알려 제어할수 있도록 돕는 기능
- 자료별 접근 통제(Label-Based Access Control) : 개별 행, 열에 대한 쓰기&읽기 권한을
  가졌는지 명확하게 결정하는 제어 방식

- 개방형 클라우드 컴퓨팅 플랫폼 : PaaS-TA (PaaS에 올라 타 의미)

- 대칭 키 암호화 : 블록 암호화 방식, DES, SEED, AES, ARIA
- 개인 키 암호화 : 스트림 암호화 방식, LFSR, RC4
- 공개 키 암호화(비대칭 암호) :  RSA

- 소프트웨어 정의 스토리지(Software Defined Storage) : 스토리지에 가상화를 적용,
  소프트웨어로 자원을 관리하여 하나의 저장장치처럼 사용할 수 있도록 하는 기법

Cocomo model
- Embeded : 초대형 규모의 시스템 소프트웨어를 대상, 30만 라인 이상의 sw개발에 적합
- Semi-Detached : Organic과 Embeded의 중간 단계, 30만 라인 이하
- Organic : 기관 내부의 중소 규모 sw대상, 5만 라인 이하 소프트웨어 개발에 적합

MQTT 프로토콜
- IBM 개발
- 발행/구독 프로토콜
- TCP/IP를 통해 실행되어 기본 네트워크 연결 제공

시스템 인증
- 토큰 기반 인증 : 하드웨어(스마트 카드)와 소프트웨어의 토큰 장치(개인 식별번호)를 사용하는 강력한 인증 방법
- SSO 인증 : 여러 시스템 중 하나의 시스템에 인증을 성공하면 다른 시스템의 접근 권한도 모두 얻는 방식

- Salt : 암호 공격을 막기 위해 똑같은 패스워드들이 다른 암호 값으로 저장되도록 추가되는 값

- OWASP(The Open Web Application Security Project) : 웹을 통한 정보 유출,
  악성 파일 및 스크립트, 보안 취약점 들을 연구하는 곳
```
