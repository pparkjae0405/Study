# 닷지 탄알 제작

트리거 콜라이더
```
충돌한 물체를 밀어내는 물리적인 표면은 없지만, 충돌 자체는 감지
충돌이 감지되었을 때 어떤 기능을 실행하는 방아쇠로 많이 사용

- 사용법
Inspector > Is Trigget 체크
```

프리팹
```
언제든지 재사용할 수 있는 미리 만들어진 게임 오브젝트 파일
비슷한 게임 오브젝트를 여러 개 만들 때 매번 다시 설정하는
번거로움을 줄이기 위해 사용

- 사용법
Hierarchy의 오브젝트를 Project로 드래그&드롭 > 
Project 창의 프리팹을 Hierarchy 창으로 드래그&드롭

- transform
자신의 게임 오브젝트의 트랜스폼 컴포넌트로 바로 접근하는 변수로
GetComponent<Transform()을 사용하여 할당하는 과정이 필요없음.

- 트랜스폼 컴포넌트
게임 오브젝트의 위치, 크기, 회전을 담당하는 컴포넌트
foward는 해당 트랜스폼을 가진 게임 오브젝트의 앞쪽 방향 제공
```

충돌 처리
```
- Destroy() 메서드
입력한 오브젝트를 파괴

- OnCollision 계열 : 일반 충돌
일반적인 콜라이더를 가진 두 게임 오브젝트가 충돌할 때 자동으로 실행
OnCollisionEnter(Collision collision) : 충돌한 순간
OnCollisionStay(Collision collision) : 충돌하는 동안
OnCollisionExit(Collision collision) : 충돌했다 분리되는 순간

- Collision 타입
충돌 관련 정보를 담아두는 정보 컨테이너


- OnTrigger 계열 : 트리거 충돌
충돌한 두 게임 오브젝트의 콜라이더 중 최소 하나가 트리거 콜라이더
일 때 자동으로 실행
OnTriggerEnter(Collider other) : 충돌한 순간
OnTriggerStay(Collider other) : 충돌하는 동안
OnTriggerExit(Collider other) : 충돌했다 분리되는 순간

- Collider 타입
트리거 충돌에는 상세한 충돌 정보가 필요 없으므로 Collider 사용

- 실제 사용
탄은 트리거 충돌 시 상대방의 태그를 확인하여 Player라면
Player의 게임 오브젝트에서 PlayerController 컴포넌트를 가져와
Die() 메서드를 실행하여 플레이어를 비활성화한다.
```

탄알 생성기
```
- FindObjectOfType() 메서드
탄알 생성기의 타겟으로 PlayerController 타입의 컴포넌트를 가진
게임 오브젝트를 찾고, 해당 오브젝트의 트랜스폼 컴포넌트를 찾아
target으로 할당

- Time.deltaTime
이전 프레임과 현재 프레임 사이의 시간 간격이 자동으로 할당

- Instantiate() 메서드
원본 오브젝트를 주면 해당 오브젝트를 복제한 오브젝트를 생성
```
