# 게임 오브젝트 제어하기

클래스와 오브젝트
```
- 클래스
묘사할 대상과 관련된 코드를 묶는 틀

- 오브젝트
클래스를 기반으로 생성한 것

- 인스턴스화
클래스라는 틀로 오브젝트를 찍어내 실체화하는 것

- 인스턴스
인스턴스화를 이용해 생성된 오브젝트
```

C# 클래스 만들기
```
- 클래스 작성
Project > C# Script > A클래스 작성

- 오브젝트 작성
Project > C# Script > A의 오브젝트 작성
Hierarchy > Create > Create Empty > 오브젝트 드래그&드롭

- 클래스의 멤버
클래스에 속하며, 해당 클래스의 데이터와 행위를 표현하는 요소

- 클래스의 필드
클래스의 멤버 중 변수

- 접근 제한자
public : 클래스 외부에서 멤버에 접근 가능
private : 클래스 내부에서만 멤버에 접근 가능
protected : 클래스 내부& 파생 클래스에서만 멤버에 접근 가능
```

참조 타입
```
- 참조 타입
클래스로 만든 변수는 참조 타입이기 때문에 실체화된 오브젝트가 아님
참조 타입의 변수를 선언하는 것만으로는 오브젝트가 생성되지 
않기 때문에 new를 사용해 오브젝트를 개별적으로 생성해야 함
클래스 a = new 클래스();를 선언했다면, a는 클래스 오브젝트를 
가리키는 참조값을 저장하는 변수

- 값 타입
float, int, string 등 C#의 내장 변수는 값 타입으로 동작함
값 타입의 변수는 해당 변수 공간에 값 자체를 저장함
```

변수로 컴포넌트 사용하기
```
Hierarchy > 3D Object > Cube
Inspector > Add Component > Rigidbody 생성
Project > C# Script > Jumper 스크립트 생성
Cube에 Jumper 스크립트 드래그&드롭
Rigidbody를 jumper의 MyRigidbody에 드래그&드롭

1. 필요한 컴포넌트(Rigidbody)를 게임 오브젝트(Inspector)에 추가
2. 스크립트(Jumper)에서 조종할 컴포넌트에 대한 변수(MyRigidbody)를 선언
3. 변수(MyRigidbody)에 컴포넌트(RigidBody)를 할당
4. 코드에서 변수를 사용(myRigidbody.AddForce(0, 500, 0);)하면 그것이 가리키는 실제 컴포넌트(AddForce();)가 동작
```
